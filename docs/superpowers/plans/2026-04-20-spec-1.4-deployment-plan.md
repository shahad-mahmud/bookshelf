# Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy Bookshelf to `bookself.iterosys.com` on free tiers with automatic CI/CD — migrations run on every Vercel deploy, PRs get a typecheck/lint/test gate via GitHub Actions.

**Architecture:** Two config files are committed to the repo: `vercel.json` overrides the Vercel build command to run `db:migrate` before `next build`, and `.github/workflows/ci.yml` runs offline quality checks on every PR. Because existing migrations were applied via `apply-migrations.ts` (which bypasses drizzle-kit), a one-time bootstrap script populates `drizzle.__drizzle_migrations` so `drizzle-kit migrate` is idempotent from the first deploy onward. Vercel connects to the existing GitHub repo natively.

**Tech Stack:** Vercel Hobby (free), GitHub Actions (free), Supabase PostgreSQL (free tier), Resend (free tier), `drizzle-kit migrate` for idempotent schema migrations, Node 20.

---

## Background: How `drizzle-kit migrate` tracks history

`drizzle-kit migrate` uses `drizzle-orm/postgres-js/migrator` under the hood. It creates a `drizzle.__drizzle_migrations` table (schema `drizzle`, table `__drizzle_migrations`) and only applies migrations whose `folderMillis` (the `when` field in `meta/_journal.json`) is greater than the latest `created_at` in that table.

Our migrations were applied via `apply-migrations.ts`, which does NOT write to this table. So on the first `drizzle-kit migrate` run against the production DB, it would see an empty table and try to re-apply all migrations — failing because the tables already exist.

**Fix (Task 3):** a one-time bootstrap script that populates `drizzle.__drizzle_migrations` with the correct SHA-256 hashes and timestamps for all already-applied migrations. After that, `drizzle-kit migrate` sees all existing migrations as done and only applies new ones.

### Task 3: Bootstrap Drizzle migration history

This script is run **once** against the production Supabase DB before the first Vercel deploy. It creates the `drizzle.__drizzle_migrations` table and inserts a record for each already-applied migration so `drizzle-kit migrate` treats them as done.

**Files:**
- Create: `scripts/bootstrap-drizzle-history.ts`
- Modify: `package.json` (add `db:bootstrap` script)

- [ ] **Step 1: Write the bootstrap script**

Create `scripts/bootstrap-drizzle-history.ts`:

```typescript
import { config as loadEnv } from 'dotenv'
loadEnv({ path: '.env.local' })
loadEnv({ path: '.env' })

import { createHash } from 'node:crypto'
import { readFileSync } from 'node:fs'
import { join } from 'node:path'
import postgres from 'postgres'

const DIRECT_URL = process.env.DIRECT_URL
if (!DIRECT_URL) throw new Error('DIRECT_URL required')

const migrationsFolder = join(process.cwd(), 'db', 'migrations')
const journal = JSON.parse(
  readFileSync(join(migrationsFolder, 'meta', '_journal.json'), 'utf8'),
)

const sql = postgres(DIRECT_URL, { max: 1, prepare: false })

async function main() {
  await sql`CREATE SCHEMA IF NOT EXISTS drizzle`
  await sql`
    CREATE TABLE IF NOT EXISTS drizzle.__drizzle_migrations (
      id   SERIAL PRIMARY KEY,
      hash TEXT   NOT NULL,
      created_at BIGINT
    )
  `

  const [{ count }] = await sql<[{ count: string }]>`
    SELECT COUNT(*) AS count FROM drizzle.__drizzle_migrations
  `
  if (Number(count) > 0) {
    console.log('Migration history already present — nothing to do.')
    await sql.end()
    return
  }

  for (const entry of journal.entries) {
    const content = readFileSync(join(migrationsFolder, `${entry.tag}.sql`), 'utf8')
    const hash = createHash('sha256').update(content).digest('hex')
    await sql`
      INSERT INTO drizzle.__drizzle_migrations (hash, created_at)
      VALUES (${hash}, ${entry.when})
    `
    console.log(`  ✓ recorded ${entry.tag}  (created_at=${entry.when})`)
  }

  console.log('Bootstrap complete. drizzle-kit migrate will only apply new migrations.')
  await sql.end()
}

main().catch((err) => {
  console.error(err)
  process.exit(1)
})
```
